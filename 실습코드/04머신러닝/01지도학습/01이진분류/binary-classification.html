이진분류가 뭐야

세상의 모든 질문을
"응" 아니면 "아니"로만 답하게 만든 문제

이진(二進)은 '둘'이라는 뜻입니다. 답이 두 종류뿐인 문제를 컴퓨터에게 맞히게 하는 거예요.

문제1이라고 부를 쪽0이라고 부를 쪽
이 메일 스팸이야?스팸정상
이 부품 불량이야?불량정상
이 손님 또 올까?재방문안 옴
이 환자 병 있어?양성음성

컴퓨터는 글자를 모르니까 한쪽을 1, 한쪽을 0으로 바꿔서 알려줍니다.

가장 먼저 정할 것 — 누가 '1'인가

이게 이진분류에서 제일 중요한 결정입니다. 그냥 아무거나 1로 정하면 안 돼요.

규칙: 찾고 싶은 것, 놓치면 큰일 나는 것을 1(양성)로 둡니다.

정확도만 보면 속습니다

환자 1000명 중 진짜 환자가 20명뿐이라고 해봅시다. 여기 아무 생각 없는 모델이 하나 있습니다.

"전부 다 정상입니다"
— 아무것도 안 보고 그냥 이렇게만 대답하는 모델

이 모델의 정확도는? 980 ÷ 1000 = 98%입니다.

한쪽이 압도적으로 많은 데이터를 불균형 데이터라고 합니다.
이럴 때 정확도는 거짓말쟁이예요. 그래서 다른 점수표가 필요합니다.

채점표 — 혼동행렬

모델의 답을 네 칸으로 나눠서 봅니다. 이 표를 혼동행렬(confusion matrix)이라고 해요.

놓치는 것(FN)과 헛다리 짚는 것(FP)은 완전히 다른 실수입니다. 병원에선 놓치는 게 치명적이고, 스팸함에선 헛다리가 더 짜증나죠.

정밀도와 재현율

이 두 가지 실수를 각각 얼마나 하는지 재는 점수입니다. 이름은 어렵지만 뜻은 간단해요.

재현율 (Recall) — "진짜 환자 중에 몇 명이나 찾아냈어?"
놓치기 싫을 때 보는 점수. 암 검사, 화재 감지, 지진 경보.

정밀도 (Precision) — "환자라고 한 것 중에 진짜가 몇 명이야?"
헛소리 하기 싫을 때 보는 점수. 스팸 필터, 상품 추천.

그래서 두 점수를 하나로 합친 게 F1입니다. 둘 다 높아야 F1이 높아지고, 한쪽만 잘해서는 점수가 안 올라가요.

기준선을 왼쪽 끝으로 밀면 놓치는 건 없지만 정상 메일이 대거 스팸함으로 갑니다. 오른쪽 끝으로 밀면 헛다리는 없지만 스팸이 다 통과해요. F1이 가장 높은 지점이 대체로 좋은 타협점입니다.

ROC-AUC — 기준선을 안 정하고 채점하기

위 실험에서 봤듯이, 점수는 기준선을 어디 두냐에 따라 계속 바뀝니다. 그럼 "이 모델 자체는 얼마나 똑똑한가?"는 어떻게 재죠?

기준선을 0%부터 100%까지 전부 시험해보고
그 성적을 통째로 하나의 숫자로 만든 것 = ROC-AUC
AUC 값뜻
1.0완벽. 스팸과 정상이 딱 갈림
0.8~0.9쓸 만함
0.5동전 던지기와 똑같음 (아무 쓸모 없음)

즉 0.5는 0점이고 1.0이 만점이라고 생각하면 됩니다. AUC가 0.5면 모델이 아무것도 못 배운 거예요.

코드로 보기

from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score, f1_score, roc_auc_score, classification_report

# 문제와 정답 나누기 (y는 0 또는 1)
y = train['is_spam']
X = train.drop(columns=['is_spam'])

# stratify=y : 나눌 때도 0과 1의 비율을 그대로 유지하라는 뜻
X_tr, X_val, y_tr, y_val = train_test_split(
    X, y, test_size=0.2, random_state=0, stratify=y)

model = RandomForestClassifier(n_estimators=100, random_state=0)
model.fit(X_tr, y_tr)

pred  = model.predict(X_val)               # 0 또는 1
proba = model.predict_proba(X_val)[:, 1]   # 1일 확률 (기준선 실험용)

print('정확도  :', accuracy_score(y_val, pred))
print('f1      :', f1_score(y_val, pred))       # 이진분류는 average 안 써도 됨
print('ROC-AUC :', roc_auc_score(y_val, proba)) # 확률을 넣어야 함! pred 아님
print(classification_report(y_val, pred))
자주 틀리는 곳 두 군데
① roc_auc_score에는 0/1이 아니라 확률을 넣어야 합니다.
② stratify=y를 빼먹으면, 데이터를 나눌 때 운 나쁘게 한쪽에 스팸이 몰릴 수 있습니다.

한 장 정리

단계할 일
1. 문제정의무엇을 1(양성)로 둘지 정한다 — 여기가 전부의 시작
2. 불균형 확인y.value_counts()로 0과 1의 개수를 본다
3. 분할stratify=y로 비율 유지하며 나눈다
4. 채점불균형이면 정확도 말고 F1·재현율·ROC-AUC
5. 기준선 조정모델을 다시 학습하지 않고도 성능을 바꿀 수 있다
이것만 기억하면 됩니다
① 정확도는 불균형 데이터에서 거짓말을 한다
② 틀리는 방식은 놓침(FN)과 헛다리(FP) 두 가지고, 상황마다 더 나쁜 쪽이 다르다
③ 기준선은 고정값이 아니라 내가 정하는 것이다
